* **字体过小或过多，阅读困难：**&#x5C06;大段文字直接复制到 PPT 上，观众根本没时间阅读。

  > * 使用要点，每点控制在 1-2 行，用短语代替完整的长句子；
  >
  > * 如果内容过多，宁可拆分成两页；
  >
  > * 能用图、表、流程图表达的，坚决不用文字。

* **缺乏统一的视觉风格：**&#x5B57;体、字号、颜色、对齐方式、Logo 位置等在不同页面随意变化，显得非常不专业。

  > * 在母版中设定好标题、正文的字体、字号、颜色和固定元素（如 Logo、页码）的位置；
  >
  > * 中文推荐使用思源黑体、微软雅黑；英文推荐 Arial, Helvetica, Calibri 等无衬线字体，更适合屏幕显示。正文字号建议不小于`18pt`。

* **颜色搭配混乱，对比度不足（文字与背景不分明）：**&#x4F7F;用了过多刺眼的颜色，或背景与文字颜色相近，导致难以辨认（尤其在投影仪上）。

  > * 全篇 PPT 需要使用一套固定的色板，颜色不超过 3-4 种（ 1 种主色，2 种辅助色）。
  >
  > * 确保文本与背景色之间有足够的对比度，以保证内容清晰易读。

* **行距、段落间距不合理，显得拥挤或空旷：**&#x884C;距太小让文字挤在一起，阅读费力；行距太大则显得空洞。

  > * 推荐正文行距为字体大小的 1.2–1.5 倍，保持整体舒适度。
  >
  > * 推荐多行的文字使用两端对齐，使段落边缘清晰，创造出更规整、易读的视觉感受。

* **图片、图表比例失调，模糊拉伸或不对齐**：直接截图或使用低质量图片，放大后出现马赛克；图表中的文字和线条模糊。

  > * 实验图表（如折线图、柱状图）尽量从 Matlab, Origin 等软件中导出为 SVG, PDF, EPS 等矢量格式，无限放大不失真；
  >
  > * 插入的照片、示意图等，保证其原始分辨率足够高。

* **图表元素混乱，重点不突出：**&#x56FE;表包含了过多的曲线、数据点、复杂的网格线，没有明确的图例和坐标轴标签。

  > * 删除不必要的背景、网格线和边框。坐标轴必须有清晰的标签和单位。
  >
  > * 确保图例位置合理，易于理解。

* **滥用动画和切换效果：**&#x4F7F;用各种旋转、飞入、弹跳等花哨的动画，分散观众注意力。

  > * 动画的唯一目的是引导视线、分步呈现，不建议使用过多动画，最好只使用“无”或“淡出”。

# 结构与逻辑类错误

* **论文质量把控不足，不标明出处：**选取的文献质量不高，缺乏代表性与前沿性，引用论文时未注明出处信息。

    > - 深入研读至少 4 篇研究论文（不含综述），至少1篇为自主检索的近 3 年重要工作；
    > - 每篇讲解的论文必须完整标明出处信息，包括作者、发表年份、论文标题、出版物名称（期刊/会议），确保引用规范、可追溯。
    > - 对于近 3 年内的工作，应优先选择顶级期刊（如 *Nature*, *Science*,  *TPAMI* ）或顶级会议（如 *CVPR*, *ICCV*, *NeurIPS*, *ICML*, *AAAI* 等）的高质量研究，保证内容的前沿性与学术水准。

* **缺乏清晰的主线和“故事性”：**&#x4ECE;头到尾平铺直叙，听众不知道重点，抓不住工作的核心脉络。

  > * 尝试用 “问题 → 现有研究方法介绍 →现有研究方法对比分析→现有的研究不足与未来方向 →结论” 的叙事逻辑串联整个汇报。
  >
  > * 明确 PPT 的整体叙事逻辑，对于内容要有自己的理解，不能只做简单的内容转述。
  >
  > * 自问 “这一页 PPT 我想传达的最关键信息是什么？” 确保每页都有一个明确的中心思想。
  >
  > * 确保PPT页面核心信息突出，让观众一目了然，迅速把握本页主旨。

* **背景介绍过长，迟迟不入正题，时间分配不合理：**&#x7531;于试图在有限时间内塞进过多细节，导致次要信息讲得太细，核心要点反而没有时间充分展开。

  > * 遵循 “1-2分钟/页” 原则，严格控制PPT页数。例如，一个 15 分钟的报告，PPT 页数最好在 10-15 页之间。
  >
  > * 花了大量时间讲众所周知的内容，导致核心方法和结果部分时间紧张。精简背景，假设听众具备基础知识，直接切入与研究最相关的“问题”或“挑战”。

* **&#x20;堆砌公式，搬运文字，缺乏自身独立的分析和解释：**&#x6EE1;屏都是复杂的数学公式，没有图解和高层次的解释，听众难以理解核心创新。

  > * 优先使用框架图、流程图来展示方法的整体结构和数据流；或者在公式旁列点进行总结性的解释。
  >
  > * 解释思想而非细节，重点解释公式背后的物理意义或设计思想，而不是推导过程。关键的公式可以放，但一定要有文字注解其作用。

* **结果展示只是罗列图表，缺乏深层解读：**&#x7F3A;乏自己对结果的深刻认识，只是简单的罗列复杂的图表或表格，然后说“结果如图所示”。

  > * 通过自己的思考和总结，在展示图表前，先用一句话概括这个实验想要验证的结论。
  >
  > * 在图表上用箭头、方框、高亮等方式，明确指出支撑结论的关键数据点。
  >
  > * 给每个图表一个有信息量的标题，例如“图1：XX方法与SOTA模型在XX数据集上的性能对比”，而不是简单的“实验结果”。

* **&#x20;草草收尾，缺乏横向对比与批判性分析：**&#x7ED3;论部分只是简单重复摘要，没有总结内容，也没有展望未来。

  > * 结尾部分用1-2页PPT，清晰地总结PPT内的主要结论，要具有概括性、总结性。
  >
  > * 体现批判性思维，从多个维度对比现有方法。
  >
  > * 深入分析现有研究的共性与趋势，权衡与取舍。
  >
  > * 最后要有引入自己的思考，比如：通过自己的思考分析，指出当前工作的局限性，并提出未来可能的研究方向，显得思考更深入、全面。

* **理论联系实际:**&#x4E0D;应止步于算法的理论分析，应当同时挖掘其在真实场景下的应用潜力。

  > * 这些前沿技术目前已经在哪些真实世界的产品或场景中得到了应用? 展示一些有趣的、具体的应用案例。
  >
  > * 展望这些技术未来可能催生哪些新的应用或改变。

* **没有过渡页，导致内容跳跃、不连贯**：主题之间缺乏承上启下的过渡，让观众感到逻辑割裂。

  > * 适度的过渡页（如章节封面、过渡标题）能让内容更连贯，也帮助观众调整思路。
